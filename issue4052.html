4052. Bogus requirement for piecewise_linear_distribution

Section: 28.5.9.6.2 [rand.dist.samp.pconst], 28.5.9.6.3 [rand.dist.samp.plinear] Status: New Submitter: Jonathan Wakely Opened: 2024-02-05 Last modified: 2024-02-05

Priority: Not Prioritized

View all other issues in [rand.dist.samp.pconst].

View all issues with New status.

Discussion:

In the second constructor of 28.5.9.6.3 [rand.dist.samp.plinear], P1719R2 replaced: 


template<class InputIteratorB, class InputIteratorW>
  piecewise_linear_distribution(InputIteratorB firstB, InputIteratorB lastB,
                                InputIteratorW firstW);

Effects: [...] Moreover, the id-expressions iterator_traits<InputIteratorB>::value_type and iterator_traits<InputIteratorW>::value_type shall each denote a type that is convertible to double.

with 

template<class InputIteratorB, class InputIteratorW>
  piecewise_linear_distribution(InputIteratorB firstB, InputIteratorB lastB,
                                InputIteratorW firstW);

Mandates: is_invocable_r_v<double, UnaryOperation&, double> is true.

Effects: [...]

This was a copy & paste error from the next constructor, and was obviously not intended. There is no UnaryOperation in that constructor.

A less obviously wrong issue is the use of double there in the first place. Shouldn't it be RealType instead? That seems to be incorrect throughout both 28.5.9.6.2 [rand.dist.samp.pconst] and 28.5.9.6.3 [rand.dist.samp.plinear], and was only partially fixed by LWG 1439.
