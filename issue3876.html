3876. INVOKE doesn't support unions since LWG 2219

Section: 22.10.4 [func.require] Status: New Submitter: Jonathan Wakely Opened: 2023-02-07 Last modified: 2023-02-07

Priority: Not Prioritized

View other active issues in [func.require].

View all other issues in [func.require].

View all issues with New status.

Discussion:

This was valid in C++14: 


#include <type_traits>
union U { int i; };
using T = std::result_of<int U::*(U&)>::type;

The expression INVOKE(declval<int U::*>(), declval<U&>()...) was well-formed, because the types met the conditions of the fourth bullet of 22.10.4 [func.require] "f is a pointer to member data of a class T and t1 is an object of type T or a reference to an object of type T or a reference to an object of a type derived from T".

LWG 2219 changed the condition to "f is a pointer to data member of a class T and is_base_of_v<T, remove_reference_t<decltype(t1)>> is true". This doesn't work because is_base_of<union U, union U> is false, for any union.

I think it should be is_same_v<T, remove_cvref_t<decltype(t1)>> || is_base_of_v<T, remove_reference_t<decltype(t1)>>
