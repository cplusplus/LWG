3914. What is the "range representable by the type" for floating-point from_chars?

Section: 22.13.3 [charconv.from.chars] Status: New Submitter: Jonathan Wakely Opened: 2023-03-29 Last modified: 2023-03-29

Priority: Not Prioritized

Discussion:

22.13.3 [charconv.from.chars] says:

If the parsed value is not in the range representable by the type of value, value is unmodified and the member ec of the return value is equal to errc::result_out_of_range.

One interpretation of this is that the parsed values for the strings "1e-10000" and "1e+10000" are outside the representable range of a double that uses IEEE binary64 format, because the min/max exponents are -1022 and +1023.

Another interpretation is that the representable range for floating-point types goes from negative infinity to positive infinity, so although there are values which cannot be represented accurately, they are still within the range of representable values. Under this interpretation, there are no values for which from_chars will leave a floating-point argument unmodified.

It would be helpful to clarify the intended meaning.
