3740. slide_view::size const should always return unsigned range_difference_t<const V>

Section: 26.7.25.2 [range.slide.view] Status: New Submitter: Hewill Kang Opened: 2022-07-15 Last modified: 2022-07-16

Priority: Not Prioritized

View other active issues in [range.slide.view].

View all other issues in [range.slide.view].

View all issues with New status.

Discussion:

Currently, slide_view::size const has the following Effects: 

auto sz = ranges::distance(base_) - n_ + 1;
if (sz > 0) sz = 0;
return to-unsigned-like(sz);

Since the types of ranges::distance(base_) and n_ are range_difference_t<const V> and range_difference_t<V> respectively, the type of sz will enter superposition states when the two are not the same, we should make sure that slide_view::size const always returns unsigned range_difference_t<const V>.

Proposed resolution:

This wording is relative to N4910.

  1. Modify 26.7.25.2 [range.slide.view] as indicated:

    constexpr auto size() requires sized_range<V>;
constexpr auto size() const requires sized_range<const V>;

    -8- Effects: Equivalent to: 

    auto sz = ranges::distance(base_) - n_ + 1;
if (sz < 0) sz = 0;
return to-unsigned-like(static_cast<decltype(ranges::distance(base_))>(sz));
