3392. Problems with ranges::distance()

Section: 23.4.3.2 [range.iter.op.distance] Status: New Submitter: Patrick Palka Opened: 2020-02-07 Last modified: 2020-02-08

Priority: Not Prioritized

View all issues with New status.

Discussion:

It is not possible to use the two-argument overload of ranges::distance() to compute the distance between a move-only counted_iterator and the default_sentinel. In other words, the following is invalid 

// iter is a counted_iterator with an move-only underlying iterator
ranges::distance(iter, default_sentinel);

and yet 

(default_sentinel - iter);

is valid.

Suggested resolution:

For the case where S models sized_sentinel_for<remove_reference_t<I>>, define a constrained overload of ranges::distance(I, S) that takes its iterator argument by const reference instead of by value (and returns last - first as before).

Proposed resolution:

This wording is relative to N4849.

  1. Modify 23.2 [iterator.synopsis], header <iterator> synopsis, as indicated:

    #include <concepts>

namespace std {
  […]
  // 23.4.3 [range.iter.ops], range iterator operations
  namespace ranges {
    […]
    // 23.4.3.2 [range.iter.op.distance], ranges::distance
    template<input_or_output_iterator I, sentinel_for<I> S>
      constexpr iter_difference_t<I> distance(I first, S last);
    template<input_or_output_iterator I, sized_sentinel_for<remove_reference_t<I>> S>
      constexpr iter_difference_t<remove_reference_t<I>> distance(const I& first, S last);
    template<range R>
      constexpr range_difference_t<R> distance(R&& r);
    […]
  }
  […]
}

  2. Modify 23.4.3.2 [range.iter.op.distance] as indicated:

    template<input_or_output_iterator I, sentinel_for<I> S>
  constexpr iter_difference_t<I> ranges::distance(I first, S last);

    -1- Preconditions: [first, last) denotes a range, or [last, first) denotes a range and S and I model same_as<S, I> && sized_sentinel_for<S, I>.

    -2- Effects: If S and I model sized_sentinel_for<S, I>, returns (last - first); otherwise, returns the number of increments needed to get from first to last. 

    template<input_or_output_iterator I, sized_sentinel_for<remove_reference_t<I>> S>
  constexpr iter_difference_t<remove_reference_t<I>> ranges::distance(const I& first, S last);

    -?- Preconditions: [first, last) denotes a range and S and I model sized_sentinel_for<S, remove_reference_t<I>>.

    -? Effects: returns (last - first);
